# Story-Specific Premium Art ExecPlan

## Purpose / Big Picture

Replace the abstract premium-book storyboard assets with story-specific generated illustrations and durable character-consistency evidence. The goal is not to raise a score by changing wording; it is to make the repo prove that every catalog story has visible, story-relevant scene art, cover art, and character-design anchors.

## Progress

- [x] Confirmed current premium art problem: 19 premium books use `local_static_storyboard_renderer` / `deterministic_repo_renderer` assets that look abstract in emulator contact sheets.
- [x] Created this ExecPlan.
- [x] Add a deterministic story-specific SVG renderer for premium scenes, covers, and character sheets.
- [x] Generate replacement PNG assets for all premium scenes/covers and all catalog character sheets.
- [x] Update manifests and character bibles with story-specific evidence fields.
- [x] Add or tighten validators so abstract placeholders cannot pass as high art quality again.
- [x] Run harness validation and inspect generated contact sheets.

## Surprises & Discoveries

- `tools/score_art_experience.py` already reports 100/100 before this pass because it checks selected asset existence, manifest status, prompts, bibles, and layer coverage. It does not currently detect that 19 premium books are visually abstract.
- The 19 premium scene PNGs are about 37 KB each and share the same abstract renderer, while the first five free-book images are much larger imported or image-generated assets.
- Pillow and CairoSVG are not installed, but macOS `sips` can convert generated SVG files to PNGs.

## Decision Log

- Keep the first five free-book scene/covers because they are already story-specific imported/image-generated assets.
- Replace only the 19 premium book scene/covers generated by the abstract `local_static_storyboard_renderer`.
- Generate character-sheet PNGs for all 24 catalog books so character consistency has file evidence, not only JSON prose.
- Keep generated assets honest as generated/reviewed internal-demo evidence, not commissioned/final production art.
- Add validation against the old abstract renderer so future scoring cannot mistake asset existence for story-specific art.

## Idempotence and Recovery

The generator is deterministic and can be rerun. It overwrites only generated-draft premium images, generated character-sheet images, related manifest metadata, reports, and character-bible evidence fields. Existing first-five story-specific assets are not replaced.
