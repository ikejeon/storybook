# Moon Jar Stories Testing Methodology Report

Generated: 2026-05-07

## Strategy

- Content contract tests: validate the 24-book catalog, 5 complete books, expanded 138-scene launch set, vocabulary/storyBeat metadata, and prompt/audio alignment.
- Story-quality tests: `tools/validate_story_quality.py` enforces read-aloud refrains, Little Listener/Story Mode text contracts, storyBeat hooks/cues/interactions, English-first audioPrompt direction, Korean optional support, and unsafe child-facing term checks.
- Native unit tests: SwiftPM XCTest and Android JVM tests assert the same shared catalog, schema contract, manifest priority, narration-track availability, and entitlement behavior the apps depend on.
- Asset validation: verify image/audio manifests, file existence, status vocabularies, review/final gates, design tokens, and voice bakeoff manifest shape.
- Readiness gates: demo/generated-draft can pass while production and release-candidate gates fail honestly until final art, final narration, store products, backend verification, and legal/store signoff exist.
- Smoke tests: backend smoke uses HTTP JSON assertions, including malformed JSON and entitlement lock/unlock cases; iOS smoke launches the app in an iPad simulator, captures screenshots, and reads app-generated self-test JSON.
- Android self-test methodology: the debug app accepts a `moonjar.selfTest=reader-contract` launch extra, resolves content/assets/narration inside the app process, writes a tokenized JSON report, and the harness rejects stale or mismatched output.
- Manual UX testing: simulator/emulator screenshots are reviewed for launch, Real Book Mode, one-spread navigation, playback state, image fit, and text layout. Android emulator smoke now passes when an emulator/device is attached.
- Review loop: local review agents found stale schema, shallow smoke tests, entitlement display gaps, and bundled shared-content drift; those findings were fixed, then a final review agent reported no local P0/P1/P2 issues.
